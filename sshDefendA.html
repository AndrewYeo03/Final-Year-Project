Exercise A: Securing SSH with Key-Based AuthenticationSuggested Duration: 10 Minutes

Learning Objectives

  • Understand the benefits of SSH key-based authentication over password-based methods.
  • Learn how to modify SSH server configurations to enforce key-based access.
  • Gain hands-on experience with securing an SSH server against brute force attacks.

Example Scenario

You are a security consultant hired by a company concerned about the security of their SSH server. During your security audit, you discover that the server allows password-based authentication, which is vulnerable to brute force attacks. The company has asked you to implement key-based authentication to strengthen server security.

Your task

Your task is to secure the SSH server by disabling password-based authentication and enabling SSH key-based authentication. This requires you to modify the SSH configuration file to ensure that only users with a valid SSH key pair can log in. After making these changes, confirm that password-based login is disabled and that key-based login works as expected.
